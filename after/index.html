Zaption Work-Sample Template

Rules

  • This is your work, and your work only. The only outside code allowed is from Knockout/jQuery/Underscore/Bootstrap, all of which are already included.
  • Google is encouraged for documentation and specific issues ("Why is Knockout throwing this error?"). But don't copy code snippets from StackOverflow, or add additional libraries. We want to see your code!
  • All of this should be done without running a server -- should all be client-side and usable by just opening index.html in my browser.
  • Make sure it works in the latest versions of Chrome.
  • Write in JavaScript. Not CoffeeScript, LiveScript, TypeScript, or other compile-to-JS languages.
  • Return this as a simple ZIP file -- not as a repo on your favorite CloudHub site!

Task

High-level summary:

Use the Zaption API to display a user's tour library back to them, using Knockout.js.

Specific requirements:

  • Get familiar with KnockoutJS (http://knockoutjs.com/), and try to write this app using Knockout best practices. The jQuery, Underscore, and Bootstrap libraries are also included (Knockout isn't for everything), but try to rely on Knockout instead of jQuery when appropriate.
  • Read up on the Zaption API (http://www.zaption.com/api). Your API client secret is removed. Pro Tip: to speed up your dev process and avoid getting hit by rate limits, you can just save an API session token and use it for development instead of sending repeated login requests.
  • Display a login form that takes a user's Zaption username/password and authenticates them using the API.
  • After authenticated, list the user's tours. If they have more tours than the API returns in a single page, just show the first page (don't worry about pagination).
  • Show a filter box that the user can type into and show only tours that match that string (client-side only, not using any Zaption API methods). This should match within words as well as full words.
  • When filtered, highlight matching parts of tour titles etc to indicate why those results were matched.

Bonus:

  • Make the page look good. This isn't a design challenge, but a little bit of CSS love goes a long way.
  • Make the code as clean as you can, and adhere to best practices for Knockout and web-development in general.
  • Implement pagination for the user's tours, using the server-side parameters.
  • Find a bug in our API!
